# 🔍 登入失敗診斷指南

## 📋 請提供以下資訊幫助診斷

### 1. 您的 Render 應用網址
請提供您的 Render 應用完整網址，例如：
`https://baccarat-prediction-xxxx.onrender.com`

### 2. 瀏覽器錯誤訊息
請按 F12 開啟開發者工具，然後：
1. 點擊 "Network" 分頁
2. 嘗試登入 (用戶名: admin, 密碼: password)
3. 查看是否有紅色的網路請求
4. 點擊紅色的請求，查看 "Response" 內容

### 3. 快速檢查步驟

#### 步驟 A: 檢查健康狀態
訪問: `https://您的網址/health`
**預期回應**:
```json
{
  "status": "OK",
  "database": "connected",
  "dbType": "postgres"
}
```

#### 步驟 B: 檢查登入頁面
訪問: `https://您的網址/login`
確認頁面能正常載入

#### 步驟 C: 檢查 Render 部署日誌
在 Render Dashboard 中查看是否有以下日誌：
```
✅ PostgreSQL 用戶表已確保存在
✅ 管理員帳號確保存在: admin / password
🚀 伺服器運行在埠 XXXX
```

## 🛠️ 可能的問題和解決方案

### 問題 1: 網路連接錯誤
**症狀**: 完全無法連接
**檢查**: `/health` 端點是否可訪問
**解決**: 可能是 Render 服務尚未啟動完成

### 問題 2: 401 未授權錯誤
**症狀**: "用戶名或密碼錯誤"
**檢查**: PostgreSQL 資料庫是否正確初始化
**解決**: 需要檢查資料庫記錄

### 問題 3: 500 伺服器錯誤
**症狀**: "內部伺服器錯誤"
**檢查**: Render 部署日誌中的錯誤訊息
**解決**: 需要修復伺服器端程式碼

3. 登入時的具體錯誤訊息 (F12 Network 面板)
4. Render 部署日誌中是否有錯誤
